## Задание 1. Имена

### Задача 3. Имена
**Текст задачи:**  
Создать сущность «Имя» с тремя полями: Фамилия, Личное имя и Отчество. Реализовать метод `toString()`, который выводит полное имя в формате «Фамилия Имя Отчество». Если какое-то поле не задано (`null`), оно не должно отображаться.  
**Алгоритм решения:**  
1. Создать класс `Name` с приватными текстовыми полями для фамилии, имени и отчества.  
2. Реализовать конструктор для инициализации всех полей.  
3. Переопределить метод `toString()`. Внутри использовать проверку каждого поля: если оно не равно `null`, добавлять его к строке результата вместе с пробелом.  
4. Применить метод `trim()` к итоговому результату, чтобы убрать лишние пробелы в начале или в конце строки.

## Задание 2. Человек

### Задача 2. Человек с именем
**Текст задачи:**  
Объединить сущности «Человек» и «Имя». Поле имени в классе «Человек» должно быть объектом типа `Name`, а не простой строкой.  
**Алгоритм решения:**  
1. В классе `Person` создать поле `name` с типом данных `Name` и поле `height` (рост).  
2. В конструкторе инициализировать объект `Name` и значение роста.  
3. В методе `toString()` вызывать встроенный метод имени (`name.toString()`) и добавлять к нему информацию о росте человека.

### Задача 3. Человек с родителем
**Текст задачи:**  
Добавить в класс «Человек» ссылку на отца (объект того же типа `Person`). Реализовать логику наследования: если у человека нет своей фамилии или отчества, они должны браться или формироваться на основе данных отца.  
**Алгоритм решения:**  
1. Добавить в `Person` поле `father`.  
2. В методе `toString()` реализовать логику проверок:  
   - Если фамилия в объекте `Name` пуста, но указан отец — использовать фамилию отца.  
   - Если отчество пусто, но указан отец — сформировать его, взяв имя отца и добавив окончание «ович».  
3. Собрать итоговую строку из полученных данных.

## Задание 3. Города

### Задача 3. Города
**Текст задачи:**  
Создать сущность «Город» с названием и списком путей в другие города. Путь включает в себя целевой город и стоимость поездки.  
**Алгоритм решения:**  
1. Создать вспомогательный класс `Path` для хранения пары «Город назначения» и «Цена».  
2. В классе `City` создать список путей `List<Path>`.  
3. Реализовать метод `addPath(City city, int cost)`, который позволяет добавлять новые маршруты из текущего города.  
4. Переопределить `toString()`, чтобы он выводил название города и список всех доступных направлений с их стоимостью.

## Задание 4. Создание объектов

### Задача 5. Создаем Имена
**Текст задачи:**  
Добавить возможность создавать объект «Имя» разными способами: только имя, имя и фамилия, или все три параметра сразу.  
**Алгоритм решения:**  
1. Использовать перегрузку конструкторов в классе `Name`.  
2. Создать три конструктора с разным набором аргументов.  
3. Те поля, которые не переданы в конструктор, будут автоматически принимать значение `null`.

### Задача 6. Создаем Человека
**Текст задачи:**  
Добавить в класс «Человек» гибкие способы создания объектов (через строку или через объект `Name`), при этом избегая дублирования кода.  
**Алгоритм решения:**  
1. Создать один основной (базовый) конструктор, который принимает `Name`, `height` и `father`. Только в нем прописать присваивание через `this.поле = значение`.  
2. Все остальные конструкторы (например, принимающие имя строкой) реализовать через вызов базового, используя конструкцию `this(...)`.  
3. Это позволит централизовать логику создания объекта в одном месте. Обновить `toString()` для вывода в формате «Имя, рост».
